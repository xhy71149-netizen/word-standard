## 💡 项目背景与核心痛点

在企业或团队的日常工作中，经常需要将来自多方的 `.docx` 或 `.doc` 文件合并汇总到一份标准的“企业红头”或是特定规范的“基准模板”中。然而，人工合并往往面临以下灾难级痛点：

- **顽固的“手动格式”**：复制粘贴带来的底色、加号、奇葩行距，在 Word 中属于“底层强制属性 (`paragraphFormat` / `font`)”，即使用了格式刷或替换样式，它们依旧如附骨之疽般盖在目标样式之上不肯离去。
- **失效的大纲与混乱的编号**：各个文件标题级别 (`Heading 1~9`) 不匹配。原本 1.1 的标题被套上原生编号，导致合并后变成几十级的混乱序列。
- **无解的“伪装 docx”死机**：部分历史遗留 `.doc` 被直接人为修改后缀为 `.docx`，结构已毁损，导致任何自动化库读取时直接爆破。

## 🚀 核心逆天特性 (Features)

本项目通过深度介入 `python-docx` 以及底层 XML (`w:pPr`, `w:rPr`, `w:vMerge`)，达成了与原生 VBA 宏 **“100% 像素级对齐”** 的高保真处理，一键解决上述所有痛点！

### 🛡️ 1. 双重“格式洗白 (Wipe Formatting)”引擎

不仅局限于应用样式名称，而是直接在底层 XML “斩草除根”。
将选定段落内所有包含手动加粗、字调色、非标准缩进的私有标签强制剔除。先将其还原至极度纯洁的 `Normal` 态，随后才为其绑定基准模板中规定的标准样式。

- **标题保护洗白**：基于 `OutlineLevel` 精准识别真正的标题（不仅靠名字），剥离伪原生编号，完美衔接至合并后的母大纲。
- **正文同构洗白**：针对全篇 `Body Text` 与 `正文`，一键剥夺各种个人编辑癖好造成的特殊偏移。

### 📊 2. 跨行“表头物理嗅探”与美化

不再依靠可怜的关键词做弱智匹配，而是**底层 XML 扫描**：
识别表格内首单元格是否存在垂直合并标签 (`w:vMerge` restart)。借此精确自动测算表头拥有 **1 行** 还是 **2 行** 的深度。
随后剥除一切自定义底纹/镶边，强力写入“单元格防换行 (`NoWrap`)”、“绝对上下居中 (`vAlign:Center`)”以及“跨表跨页重复 (`tblHeader`)”属性，让所有业务表格瞬间排向阅兵式。

### 🤖 3. Win32COM 终极防空洞 (抗崩溃转码机制)

针对结构损坏的畸形文档以及假 `.docx` 改名文件，引入自动静默后台修复机制：
一旦主线解析崩溃，**自动在无感后台唤醒本机正版 Office Word 进程**，以只读加静默模式强吃该文件，并在一秒内借由微软的官方解析器重铸为全新的纯正格式缓存进行二次合并！不弹窗，不死机，不闪退！

### 🖼️ 4. 浮动图片的强力捕获

精准遍历包含形体（`<w:drawing>`）的段落，无视其原本所处的漂浮环境，一律重置设定其目标物理尺寸并打上“图文”定点样式进行绝对居中。

## 📦 如何使用 (Usage)

由于目前已将完整的图形化界面打包成了单文件（Standalone Executable），在任何 Windows 设备上，只需双击运行，不再需要配置任何环境！

1. **部署执行**：双击运行 `Word快速标准化助手.exe`（界面请全屏运行）。
2. **三步走操作**：
   - 🎯 **第 1 步【选择基准模板】**：即“规矩”，里面包含想要最终达成的标题/正文/表格样式的标准红头样式空文档。
   - 📄 **第 2 步【选择文档矩阵】**：拖曳或勾选你需要一并洗白、合并的所有乱格式标的物文件。
   - 🚀 **第 3 步【精细控制与合并】**：在下方勾选你需要的高级清理功能（标题/正文/表格/图文），点击绿色的执行按钮。
3. **享受成果**：后台即刻多重清洗后，弹窗通知您取走新鲜出炉的标准融合版文档。
